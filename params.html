PARAMS Reference

All values are set in the PARAMS dict in nse_screener.py. Click section headers to collapse.

Stage 1 Pre-Filter
ParameterDefaultDescription
min_price $5.00 Skip penny stocks and distressed micro-caps. Below this price, liquidity and data quality are unreliable.
s1_rsi_min 40 Lower RSI bound for Stage 1. Wide net — allows early movers not yet at 50.
s1_rsi_max 85 Upper RSI bound for Stage 1. Very generous — extreme overbought cases only excluded.
s1_volume_ratio_min 1.2× Minimum above-average volume interest required to pass Stage 1. Excludes dead volume days.
s1_close_vs_high_min 0.96 Close must be at least 96% of the day's high. Eliminates collapsing candles from consideration.
s1_ma20_proximity 0.95 Close must be at least 95% of the 20-day MA. Requires the stock to be near its short-term trend.
Stage 2 BUY Rules
ParameterDefaultDescription
close_vs_high_min 0.985 Close must be at least 98.5% of the day's high — tight gate requiring buyers to hold price near top.
volume_ratio_min 1.5× Volume must exceed 1.5× the 20-day average. Institutional-level conviction threshold.
rsi_min 50 RSI must be above 50 — confirms momentum has crossed into bullish territory.
rsi_max 75 RSI must be below 75 — avoids entering already-overbought setups.
macd_line_positive True MACD line (EMA12 − EMA26) must be positive. Confirms medium-term trend is up.
close_above_ma20 True Close must be above the 20-day moving average — short-term trend confirmed.
bullish_candle True Close must be greater than Open — green candle required.
w52_high_cap 99.0% Block signals within 1% of 52-week high unless a Bollinger Band breakout confirms.
earnings_buffer_days 5 Reject signals if earnings are within ±5 calendar days. Removes binary event risk.
max_debt_equity 300 Reject if Debt/Equity ratio exceeds 300 on yfinance's raw scale. Filters extreme leverage.
Stage 2 SELL Rules
ParameterDefaultDescription
sell_rsi_min 30 SELL will not fire if RSI is below 30. Avoids entering into oversold bounce traps.
sell_rsi_max 55 SELL will not fire if RSI is above 55. A reading above 55 indicates momentum still intact — not a SELL setup.
sell_cooldown_days 5 No repeat SELL on the same symbol within 5 calendar days. Shorter window than BUY cooldown.
buy_sell_conflict_window 5 Block a SELL signal if a BUY fired in the last 5 days (and vice versa). Prevents contradictory signals.
Signal Quality & Conviction
ParameterDefaultDescription
noise_floor_score 30 Any signal scoring below 30 is dropped entirely. Eliminates ~42% of historical low-quality output.
etf_min_score 40 ETFs require a minimum score of 40. ETFs lack fundamental data (no EPS, D/E), so technicals must compensate.
score_high 75 Signals scoring 75+ receive HIGH conviction label. Fire in all market regimes including NEUTRAL.
score_medium 55 Signals scoring 55–74 receive MEDIUM conviction label. Suppressed in NEUTRAL regime.
signal_cooldown_days 20 No repeat BUY signal on the same symbol within 20 calendar days. Prevents over-signalling.

Symbol Filter Lists

Two hardcoded lists govern special handling for inverse ETFs and ETFs.

SELL_EXCLUSION_SYMBOLS — Inverse/Volatility ETFs (never trigger SELL)
A SELL on SQQQ means "bullish on QQQ" — logically inverted. These symbols are always blocked from the SELL signal path.
SQQQSPXSSPXUUVXYVXX SOXSTZAFAZSDOWSRTY VIXYSVXY
ETF_SYMBOLS — Higher minimum score required (40+)
ETFs have no Forward PE / EPS growth data. The fundamental scoring bonus is unavailable, so a higher technical score is required.
SPYQQQDIAIWMVTI VOOTQQQGLDSLVGDX TLTIEFXLKXLFXLV XLPXLEXLIXLCXLRE IBBKREVFHVNQIYR REMHACKBUGSMHSOXX ITA

CLI Reference

Run the screener manually from the command line with full flag control.

bash screener/nse_screener.py
python screener/nse_screener.py \
  --market us \
  --mode daily \
  --period 6mo \
  --symbols AAPL MSFT NVDA TSLA \
  --output signals.csv
Flag Options Default Description
--market us · canada · india · india_bse india Market to scan. Sets symbol suffix (.NS, .TO, etc.) and currency.
--mode daily · week · friday week daily = today's signal only. week = Mon–Fri window. friday = end-of-week scan.
--period 3mo · 6mo · 1y · 2y 6mo yfinance history period. Longer periods = more accurate MAs but slower download.
--skip-fundamentals (flag) off Skip Stage 2 fundamental enrichment. Useful for fast test runs — no extra API calls.

GitHub Secrets

All credentials are stored as GitHub Actions repository secrets — never committed to code.

GMAIL_ADDRESS
Sender Gmail address (e.g. you@gmail.com). Used as the From address for all alert and validation emails.
GMAIL_APP_PASSWORD
16-character Gmail App Password — not your account login password. Enable 2FA → myaccount.google.com/apppasswords → create "Stock Screener" → copy the code.
RECIPIENT_EMAIL
Email address where all signal alerts and validation reports are delivered. Can be different from the sender address.
DASHBOARD_TOKEN
GitHub Personal Access Token with repo scope. Used by the workflow to push generated HTML to the public dashboard repository.
