How It Works

Six automated steps run every weekday at 6 PM EST via GitHub Actions — no servers, no subscriptions.

1
GitHub Actions Triggers
Scheduled cron job fires Monday–Friday at 6 PM EST (11 PM UTC). Also triggerable manually via the Actions UI.
cron: 0 23 * * 1-5
2
yfinance Downloads OHLCV
300+ symbols across 13 sectors, 6 months of daily history. Free API — no key required. Flattens MultiIndex columns automatically.
300+ symbols · 6mo history
3
Stage 1 — Technical Pre-Filter
Fast gate: RSI 40–85, Vol Ratio ≥ 1.2×, Close/High ≥ 0.96, price above 95% of MA20. Roughly 15–30% of symbols pass.
~15–30% pass rate
4
Stage 2 — Strict Gates + Fundamentals
All-or-nothing: RSI 50–75, MACD line > 0, Vol ≥ 1.5×, Close/High ≥ 98.5%, bullish candle, earnings buffer, D/E cap. Fundamental data fetched via yfinance info.
9 gates — all must pass
5
Conviction Scoring (0–100)
Five categories: MACD state, RSI zone, volume tier, price action, trend structure. Bonus points for fundamental quality. Labels: HIGH (≥ 75) · MEDIUM (≥ 55) · SPECULATIVE (< 55).
5 categories · max 100 pts
6
Email + Dashboard + Git Commit
HTML email sent via Gmail SMTP. GitHub Pages dashboard updated. Results committed to results/YYYY-MM-DD.csv for historical tracking.
email · pages · git

Two Outputs, Every Run

Every weekday run produces an email alert and updates the live dashboard automatically.

📧
Email Alert
HTML-formatted email sent via Gmail SMTP after every run. Color-coded conviction badges (HIGH / MEDIUM / SPECULATIVE), signal tables with Close, RSI, MACD, Volume Ratio, and 52W High columns. Sends "no signals found" confirmation on quiet days.
📊
Live Dashboard
Auto-generated HTML page hosted on GitHub Pages. Updated after every run. Shows the latest signals with full metadata. Historical archive pages available at docs/YYYY-MM-DD.html.
