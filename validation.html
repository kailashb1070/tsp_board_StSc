How Validation Works

Four automated steps run at 11:30 PM EST daily via validate.yml.

1
Find File
Looks for results file from ~5 trading days ago in results/. Accepts ±1 day tolerance for holidays.
2
Fetch Prices
yfinance fetches current closing price for each signal symbol. Free, no API key required.
3
Compute Returns
(current − signal) / signal × 100 — percentage return over the 5-day window.
4
Score Accuracy
BUY correct if price rose. SELL correct if price fell. Overall + per-type accuracy computed.
// accuracy definition

What Counts as Correct

Simple directional accuracy — did the price move in the signalled direction over the 5-day window?

BUY Correct
Current price is higher than the signal price after 5 trading days. Stock moved up as the BUY signal predicted.
current_price > signal_price
SELL Correct
Current price is lower than the signal price after 5 trading days. Stock moved down as the SELL signal predicted.
current_price < signal_price
No Data
Price unavailable from yfinance (delisted, halted, or API gap). This symbol is excluded from the accuracy denominator — not counted as wrong.

Accuracy Interpretation

How to read the overall accuracy number that appears in validation emails.

≥ 60%
Above Random
Screener is performing above random (50%). Signal logic is working. No action needed.
40–60%
Within Noise
Within statistical noise. Monitor over several weeks before drawing conclusions about PARAMS.
< 40%
Review Required
Below random consistently. Review PARAMS thresholds. Consider regime mismatch or data quality issue.
